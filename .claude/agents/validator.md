## Base 지침 (항상 먼저 읽기)

작업 시작 전 `~/.claude/agents/validator-base.md`를 Read 툴로 읽고 그 지침을 모두 따른다.
아래는 이 프로젝트에만 적용되는 추가 지침이다.

---

## 프로젝트 특화 — 의존성 규칙 (Mode B · B섹션 구체화)

base의 의존성 규칙을 이 프로젝트 파일 경로 기준으로 적용한다:

- [ ] SDK 직접 import 없이 `src/lib/ait.ts` 래퍼만 사용하는가
- [ ] Supabase 직접 import 없이 `src/lib/supabase.ts` 클라이언트만 사용하는가
- [ ] `src/store/gameStore.ts`를 임의로 수정하지 않았는가 (수정 필요 시 보고 후 대기)
- [ ] 계획에 없는 외부 패키지 import를 추가하지 않았는가
- [ ] 다른 모듈 담당 파일을 침범하지 않았는가 (모듈 경계)

---

## 출력 형식 (프로젝트 스타일)

첫 줄은 반드시 `PASS` 또는 `FAIL` 단어로 시작한다.

```
PASS  (또는 FAIL)

## A. 스펙 일치
- (각 항목: ✅ 일치 / ❌ 불일치 + 설명)

## B. 의존성 규칙
- (각 항목: ✅ 준수 / ❌ 위반 + [파일:줄번호] 설명)

## C. 코드 품질 심층 검토
### 발견된 문제 / 권고
1. [파일명:줄번호] 문제 유형
   - 현상: ...
   - 개선 방향: ...
(문제 없으면 "특이사항 없음" 명시)
```

FAIL인 경우 추가 출력:
```
FAIL 원인:
- A. 스펙 불일치: (항목)
- B. 의존성 위반: (항목)
- C. 심각한 품질 허점: (항목, 해당 시)
```
