# 03-app-subtitle-update — 앱 노출 부제 업데이트

> Story 3 / Issue #137
> 관련 에픽: Epic 13 (#134)
> **주의**: 부제 최종 문구는 유저(운영) 확인 후 확정. 이 impl은 파일 경로·변경 방법만 명시한다.

---

## 목적

F5 폐기로 "기억력으로 토스포인트 받으세요" 부제가 유효하지 않다.
앱인토스 콘솔 노출 문구와 `granite.config.ts` 내 관련 필드를 새 문구로 교체한다.

---

## 현황 파악

`granite.config.ts`를 확인한 결과, 현재 `displayName`만 정의되어 있고 `description`·`subtitle` 필드는 없다.

```typescript
// granite.config.ts 현재 상태
export default defineConfig({
  appName: 'memory-battle',
  brand: {
    displayName: '기억력배틀',
    primaryColor: '#ff6900',
    icon: '...',
  },
  // ...
})
```

앱인토스 부제(부가 설명)는 **앱인토스 콘솔 노출 정보**에서 관리한다. granite.config.ts의 코드 변경이 필요한지는 콘솔에서 먼저 확인 필요.

---

## 수정 대상 (문구 확정 후 적용)

### A. 앱인토스 콘솔 (운영 작업 — 코드 변경 없음)

| 항목 | 현재 값 | 변경 방향 |
|---|---|---|
| 앱 부제 | "기억력으로 토스포인트 받으세요" | 아래 옵션 중 확정 후 적용 |

### B. granite.config.ts (옵션에 따라 코드 변경 발생 가능)

현재 `brand.description` 필드가 존재하지 않아 코드 변경 없이 콘솔에서만 수정 가능. 만약 granite.config에 description 필드가 추가된다면:

```typescript
// 변경 파일: granite.config.ts
brand: {
  displayName: '기억력배틀',
  description: '[확정 문구]',  // 추가 시
  primaryColor: '#ff6900',
  icon: '...',
}
```

### C. RELEASE.md 업데이트 (문구 확정 후)

```markdown
## 앱 노출 정보 업데이트 (v0.5)
- [ ] 앱인토스 콘솔 > 앱 노출 정보 > 부제 변경
      현재: "기억력으로 토스포인트 받으세요"
      변경: "[확정 문구]"
```

> 최종 문구는 유저(운영 담당자) 확인 후 확정. 확정 전까지 이 impl 파일 실행 보류.

## 결정 근거

**운영 작업 분리**: 부제는 앱인토스 콘솔에서 관리하는 노출 메타데이터다. granite.config.ts에 description 필드가 없으므로 코드 커밋 없이 콘솔에서만 변경 가능한 경우가 많다. 코드 변경이 필요한 경우에만 PR을 만들면 된다.

**문구 TBD**: PRD §9에서 명시적으로 TBD 처리됨. 아키텍트가 문구를 임의 확정하지 않는다. 파일 경로와 변경 방법만 명시하고 문구는 유저 결정 대기.
