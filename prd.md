## 3. 게임 오버 광고/코인 흐름 ⚠️ v0.4 변경

> **기존 데일리 10포인트 폐지** — 코인 지급 방식으로 전환.
> `daily_reward` 테이블 및 관련 로직 전부 제거.

| 단계 | 행동 | 결과 |
|---|---|---|
| 게임 오버 | 리워드 광고 자동 시작 (강제) | 스킵 불가 상태로 광고 시작 |
| 완시청 (userEarnedReward) | 코인 랜덤 지급 | 1~5개 랜덤 (가중치, 횟수 제한 없음) |
| 스킵 / 광고 실패 | 코인 미지급 | 다시하기 버튼 활성화 |
| 다시하기 버튼 | 누르면 게임 재시작 | 플레이 횟수 제한 없음 |

**광고 보상 코인 랜덤 테이블**:

| 지급량 | 확률 |
|---|---|
| 1개 | 30% |
| 2개 | 30% |
| 3개 | 25% |
| 4개 | 10% |
| 5개 | 5% |

> 평균 기댓값 ≈ 2.3개/판. 일일·횟수 제한 없음.

## 7. 화면 목록 (MVP) ⚠️ v0.4 변경

1. **메인 화면** — 게임 시작 버튼 + 내 랭킹 순위 + **코인 잔액 표시**
2. **게임 화면** — 4개 버튼 + 현재 점수 + 콤보 배율 표시 + 배너광고 + 타임워치
3. **게임 오버 화면**
   - 이번 게임 점수 / 최고 도달 스테이지 / 풀콤보 달성 횟수 + 최고 콤보 스택 + 콤보 보너스 점수
   - 랭킹 진입 시: 일간 / 월간 / 시즌 각 순위 표시
   - **부활 버튼** (5코인): 잔액 부족 → disabled + "코인이 부족합니다 (N개)" / 이미 부활 → disabled + "이미 사용했습니다"
   - 리워드 광고 자동 시작 → 완시청 → "🪙 코인 N개 획득!" + 현재 잔액 표시
   - **토스포인트 교환 버튼** (10코인=10포인트): 잔액 부족 → disabled + "코인 10개가 필요합니다 (N개)"
   - 광고 종료 후 **다시하기** 버튼 활성화
4. **랭킹 화면** — 일간 / 월간 / 시즌 탭 + TOP 50 + 내 순위 하단 고정

## 11. 코인 시스템 UX 흐름

```
메인화면 [🪙 잔액]
  → (게임 시작) → 게임화면
  → (게임오버) → 게임오버화면

게임오버화면 [🪙 현재 잔액]
  → (광고 자동 시작) → 완시청 → "🪙 +N 획득!" 피드백
  → (최고기록 갱신 시) → "🏆 +1 코인" 피드백
  → [부활 버튼: 5코인] 활성 → 5코인 차감 → 게임화면(현 스테이지, score/combo 유지)
  → [교환 버튼: 10코인] 활성 → SDK 호출 → "10포인트 지급!" 피드백
  → (다시하기) → 메인화면
```

**부활 상태머신 (v0.4 추가):**
```
INPUT → 오답/타임아웃 → GameOverOverlay
  ├─ [balance≥5, revivalUsed=false] revive()
  │    → sequence=[] 초기화 (같은 stage 길이로 새 시퀀스)
  │    → score / comboStreak / stage 유지
  │    → status='SHOWING'
  └─ [결과 보기 or revivalUsed=true] → RESULT (부활 버튼 없음)
```

**store 추가 필드:**
```typescript
revivalUsed: boolean  // 판당 1회 제한. resetGame() 시 false 초기화
```

## 12. 기능별 수용 기준 (Given / When / Then)

### F1 — 코인 DB 인프라 & useCoin 훅

| 시나리오 | Given | When | Then |
|---|---|---|---|
| 신규 유저 코인 적립 | user_coins row 없음 | add_coins(userId, 2) | balance=2 row 생성 |
| 잔액 음수 방지 | balance=3 | add_coins(userId, -5) | balance=0 (음수 차단) |
| 동시 적립 atomic | balance=0 | add_coins(+3), add_coins(+2) 동시 | balance=5 |
| 네트워크 오류 fallback | getBalance 실패 | 앱 진입 | 잔액 0 표시, 게임 차단 안 함 |

### F2 — 광고 완시청 코인 적립

| 시나리오 | Given | When | Then |
|---|---|---|---|
| 완시청 코인 지급 | 광고 완시청 | userEarnedReward 콜백 | 1~5개 코인 + "🪙 N개 획득!" 토스트 |
| 광고 스킵 | 광고 스킵 | 광고 종료 | 코인 미지급, 다시하기 활성화 |
| 샌드박스 고정 | 샌드박스 환경 | 완시청 | 정확히 2개 지급 |

### F3 — 최고기록 코인 보상

| 시나리오 | Given | When | Then |
|---|---|---|---|
| 최고기록 갱신 | prevBest=250 | score=300 | +1코인 + "🏆 최고기록! 🪙 +1개" 토스트 |
| 동점 미처리 | prevBest=250 | score=250 | 코인 미지급 |
| 첫 플레이 | prevBest=0 | score=50 | +1코인 |

### F4 — 부활 아이템

| 시나리오 | Given | When | Then |
|---|---|---|---|
| 부활 성공 | balance=7, revivalUsed=false | 부활 탭 | balance=2, revivalUsed=true, SHOWING 재진입 |
| 코인 부족 | balance=4 | 게임오버 오버레이 진입 | 부활 버튼 비활성 + "코인이 부족합니다 (현재 4개)" |
| 이미 사용 | revivalUsed=true | 게임오버 오버레이 진입 | 부활 버튼 비활성 + "이미 부활을 사용했습니다" |
| 부활 후 게임오버 | revivalUsed=true 상태 | ResultPage 진입 | 부활 버튼 미표시 |
| 부활 점수 랭킹 반영 | 부활 후 게임오버 | scores INSERT | 랭킹 정상 반영 |

### F5 — 토스포인트 교환

| 시나리오 | Given | When | Then |
|---|---|---|---|
| 교환 성공 | balance=15 | 교환 탭 | balance=5, 10포인트 지급, "🎉 10포인트 지급됐어요!" |
| 코인 부족 | balance=9 | 교환 버튼 진입 | 버튼 비활성 + "코인 10개가 필요합니다 (현재 9개)" |
| SDK 오류 롤백 | grantPromotionReward 실패 | 교환 시도 | add_coins(+10) 롤백, 오류 토스트 |

### F6 — daily_reward 코드 제거

| 시나리오 | Given | When | Then |
|---|---|---|---|
| 타입 에러 없음 | 코드 제거 완료 | npx tsc --noEmit | 컴파일 에러 0건 |
| 앱 정상 동작 | 코드 제거 완료 | 앱 실행 | 광고 흐름 정상 동작 |
| 참조 코드 없음 | 전수 검색 | daily_reward 검색 | DB 스키마 주석 외 0건 |
